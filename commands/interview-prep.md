## Interview Prep Structure

### Phase 1: Assessment (Day 1)
- Quick skill evaluation
- Identify weak areas
- Set realistic goals
- Create personalized plan

### Phase 2: Foundation (Weeks 1-2)
**Topics**: Foundations, Linear Structures
- [ ] Big O notation mastery
- [ ] Array operations (MUST do: Two Sum, Best Time to Buy Stock)
- [ ] Linked lists (MUST do: Reverse, Detect Cycle)
- [ ] String manipulation (MUST do: Palindrome, Anagrams)
- [ ] Stack/Queue basics (MUST do: Valid Parentheses)

**Daily**: 2-3 problems + concept review

### Phase 3: Core Structures (Weeks 3-4)
**Topics**: Searching, Sorting, Non-Linear Structures
- [ ] Binary search mastery
- [ ] Sorting algorithms (understand, not memorize)
- [ ] Trees fundamentals (MUST do: Inorder/Preorder/Postorder)
- [ ] BST operations (MUST do: Validate BST, Lowest Common Ancestor)
- [ ] Heaps and priority queues

**Daily**: 3-4 problems + mock explanations

### Phase 4: Advanced (Weeks 5-6)
**Topics**: Dynamic Programming, Graphs
- [ ] DP problem identification
- [ ] Common DP patterns (MUST do: Longest Increasing Subsequence, Coin Change)
- [ ] Graph traversal (MUST do: BFS, DFS, Connected Components)
- [ ] Shortest path (MUST do: Dijkstra, Bellman-Ford)
- [ ] Minimum spanning trees

**Daily**: 4-5 problems + system design basics

### Phase 5: Specialization (Weeks 7-8)
**Topics**: Advanced Topics, Optimization
- [ ] Company-specific patterns
- [ ] Bit manipulation tricks
- [ ] Hash table deep dive
- [ ] Advanced data structures
- [ ] System design questions

**Daily**: 5 problems + mock interviews

## Company-Specific Guides

### Google/Facebook/Amazon/Microsoft
- **Round 1**: 2 medium problems (45 min)
- **Round 2**: 1 medium + 1 hard (60 min)
- **Onsite**: 4-5 technical rounds
- **Focus**: DSA + System Design

**Key Topics**:
- Arrays, strings, linked lists
- Trees and graphs
- Dynamic programming
- Basic system design

### Startups
- **Focus**: Speed + fundamentals
- **Rounds**: 1-2 technical
- **Emphasis**: Practical problem-solving
- **Less focus**: Advanced optimization

### Interview Patterns by Company

#### Google Style
- Expect hard problems
- Need perfect solution
- Follow-up questions common
- System design may appear

#### Amazon Style
- Behavioral + Technical
- Medium difficulty mainly
- Leadership principles matter
- 4-5 round loop

#### Microsoft Style
- Multiple coding questions
- Focus on code quality
- Design/patterns important
- Behavioral integrated

## Daily Checklist

### Before Interview Day
- [ ] Sleep 8 hours
- [ ] Eat good breakfast
- [ ] Test technical setup
- [ ] Prepare environment
- [ ] Review key formulas/patterns
- [ ] Calm mind exercises

### During Interview (45 min)
- [ ] **Clarify (3 min)**: Ask questions, understand problem
- [ ] **Approach (2 min)**: Outline solution before coding
- [ ] **Code (25 min)**: Write clean, tested code
- [ ] **Test (5 min)**: Walk through examples
- [ ] **Optimize (5 min)**: Discuss complexity, improvements
- [ ] **Questions (5 min)**: Ask about their work/culture

## Time Management

### 45-Minute Interview
- 2 min: Read and understand
- 5 min: Discuss approach
- 30 min: Code and test
- 5 min: Optimize
- 3 min: Questions

### 60-Minute Interview
- 3 min: Read and understand
- 7 min: Discuss approach
- 35 min: Code and test
- 10 min: Optimize
- 5 min: Questions

### Onsite (Multiple rounds)
- Similar time distribution
- Difficulty increases
- Last rounds often system design
- Behavioral mixed throughout
