## Interview Strategy

### Phase 1: Clarification (2 minutes)
```
Ask interviewer:
1. "What's the input size?" ‚Üí Algorithm selection
2. "What are constraints?" ‚Üí Time/memory limits
3. "Can I modify input?" ‚Üí In-place vs extra space
4. "What about duplicates/nulls?" ‚Üí Edge cases
5. "Need to optimize further?" ‚Üí Follow-up
```

### Phase 2: Planning (5-8 minutes)
```
Explain approach:
- "I'll use two pointers because..."
- "Time complexity will be O(...)"
- "Space complexity will be O(...)"
- Get interviewer approval before coding
```

### Phase 3: Implementation (15-20 minutes)
```
Code clearly:
- Write readable variable names
- Add comments for complex logic
- Don't skip edge cases
- Test as you code
```

### Phase 4: Testing (5-10 minutes)
```
Test with:
1. Simple example (n=1 or n=2)
2. Expected case (from problem)
3. Edge case (empty, duplicates, negative)
4. Large case (maximum input)
```

### Phase 5: Optimization (5 minutes)
```
Discuss:
- "Can we optimize time?" ‚Üí O(n¬≤) ‚Üí O(n log n)
- "Can we reduce space?" ‚Üí O(n) ‚Üí O(1)
- "Are there tradeoffs?" ‚Üí Time vs space
```

## Interview Tips

### Before Interview
- [ ] Know your strengths and weaknesses
- [ ] Practice 30 problems minimum
- [ ] Get good sleep night before
- [ ] Test your setup (camera, mic, internet)
- [ ] Arrive 5 minutes early

### During Interview
- [ ] Listen carefully to problem
- [ ] Ask clarifying questions
- [ ] Think aloud (interviewer sees process)
- [ ] Don't jump into coding
- [ ] Test your solution

### After Interview
- [ ] Send thank you note
- [ ] Mention positive interactions
- [ ] Ask about next steps
- [ ] Follow up in one week if no response

## Common Mistakes

‚ùå **Starting to code immediately**

‚ùå **Ignoring edge cases**

‚ùå **Not thinking aloud**

‚ùå **Pessimistic about optimization**

‚ùå **Poor code quality**
